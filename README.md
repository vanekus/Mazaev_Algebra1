2. Математические формулы методов
2.1. Метод Гаусса (прямой и обратный ход)
Рассматривается система A x = b с квадратной матрицей A размера n × n.

Прямой ход (приведение к верхнетреугольному виду).
Для k = 0, 1, ..., n-1:

Проверка: если |a(k,k)| < ε, матрица вырождена (для метода без выбора).

Для каждой строки i = k+1, ..., n-1 вычисляется множитель:


m(i,k) = a(i,k) / a(k,k)
Элемент a(i,k) обнуляется, а остальные элементы строки i преобразуются:


a(i,j) = a(i,j) - m(i,k) * a(k,j)   для j = k+1, ..., n-1
Одновременно преобразуется правая часть:


b(i) = b(i) - m(i,k) * b(k)
В результате получается верхнетреугольная матрица U.

Обратный ход (нахождение решения).
Для i = n-1, n-2, ..., 0:



         b(i) - Σ_{j=i+1}^{n-1} u(i,j) * x(j)
x(i) = -----------------------------------------
                        u(i,i)
Частичный выбор ведущего элемента: на каждом шаге k ищется строка p ∈ [k, n-1] с максимальным |a(p,k)|, и она меняется местами со строкой k перед исключением.

2.2. LU-разложение (метод Дулитла)
Матрица A факторизуется в виде A = L·U, где
L – нижнетреугольная матрица с единичной диагональю (l(i,i) = 1),
U – верхнетреугольная матрица.

Компактная схема (без перестановок):
Для k = 0, ..., n-1:

Строка U: u(k,j) = a(k,j) - Σ_{s=0}^{k-1} l(k,s)·u(s,j) для j = k, ..., n-1.

Столбец L: l(i,k) = ( a(i,k) - Σ_{s=0}^{k-1} l(i,s)·u(s,k) ) / u(k,k) для i = k+1, ..., n-1.

В программной реализации множители l(i,k) хранятся под главной диагональю, а u(k,j) – на диагонали и выше.

Решение системы с LU-разложением сводится к двум треугольным подстановкам:

Прямая подстановка (L y = b):


y(i) = b(i) - Σ_{j=0}^{i-1} l(i,j) * y(j)   (i = 0..n-1, деление не требуется, т.к. l(i,i)=1)
Обратная подстановка (U x = y):


x(i) = ( y(i) - Σ_{j=i+1}^{n-1} u(i,j) * x(j) ) / u(i,i)   (i = n-1..0)
3. Описание реализованных методов
3.1. Метод Гаусса без выбора ведущего элемента (gaussNoPivot)
Идея: Последовательное исключение неизвестных путём вычитания ведущей строки, умноженной на множитель, из нижележащих строк.

Особенности: Прост, но неустойчив при появлении малых ведущих элементов.

Сложность: O(n³) для прямого хода, O(n²) для обратного.

3.2. Метод Гаусса с частичным выбором ведущего элемента (gaussPartialPivot)
Идея: Перед исключением в столбце k выбирается строка с максимальным по модулю элементом и меняется с текущей. Это снижает влияние ошибок округления.

Особенности: Небольшие дополнительные затраты на поиск максимума и перестановку строк.

Сложность: O(n³).

3.3. LU-разложение (luDecompose)
Идея: Преобразование матрицы A в произведение L·U путём модификации прямого хода Гаусса, при котором множители сохраняются на месте обнуляемых элементов.

Особенности: Выполняется без выбора ведущего элемента, поэтому чувствительно к нулевым диагональным элементам. Результат хранится компактно в одной матрице.

Сложность: O(n³).

3.4. Прямая подстановка (forwardSubstitution)
Решает L·y = b, где L – нижнетреугольная с единицами на диагонали.

Алгоритм: y(i) = b(i) - Σ_{j< i} L(i,j)·y(j). Деление не требуется.

Сложность: O(n²).

3.5. Обратная подстановка (backwardSubstitution)
Решает U·x = y, где U – верхнетреугольная.

Алгоритм: x(i) = ( y(i) - Σ_{j>i} U(i,j)·x(j) ) / U(i,i).

Сложность: O(n²).

3.6. Решение через LU (solveLU)
Обёртка над прямой и обратной подстановками для готового LU-разложения.

1. Все методы имеют кубическую сложность, время работы сопоставимо. Выбор ведущего элемента даёт небольшие накладные расходы.
2. При решении многих систем с одной матрицей LU-разложение даёт многократный выигрыш (до 34 раз при k = 100) за счёт однократного разложения.
3. На плохо обусловленной матрице Гильберта методы без выбора ведущего элемента теряют устойчивость (при n ≥ 10). Частичный выбор сохраняет работоспособность, но точность падает с ростом n.
