
 Отчёт по лабораторной работе  
Сравнение методов решения СЛАУ: метод Гаусса и LU-разложение

1. Постановка задачи
Реализовать методы решения СЛАУ:
- метод Гаусса без выбора ведущего элемента,
- метод Гаусса с частичным выбором ведущего элемента по столбцу,
- LU-разложение (компактная схема Дулитла).

Провести сравнение времени работы для однократного решения (n = 100, 200, 500, 1000), показать преимущество LU-разложения при многих правых частях (n = 500, k = 1, 10, 100) и оценить точность на матрице Гильберта (n = 5, 10, 15).

2. Краткое описание алгоритмов
- Метод Гаусса: прямой ход приводит матрицу к верхнетреугольному виду исключением поддиагональных элементов, обратный ход находит решение.  
  Частичный выбор– перед исключением в столбце ищется максимальный по модулю элемент и строки меняются местами для повышения устойчивости.
- LU-разложение: матрица A представляется как L * U (L - нижнетреугольная с единичной диагональю, U - верхнетреугольная). Решение сводится к двум треугольным подстановкам: L*y = b, затем U*x = y. Разложение выполняется один раз, подстановки - быстро.

 3. Структура программы
- matrix.h/cpp - класс Matrix (хранение, доступ, генерация, умножение, нормы).
- solver.h/cpp - статические методы Solver с реализацией алгоритмов.
- timer.h- замер времени через std::chrono.
- experiment.h/cpp- проведение трёх экспериментов и вывод таблиц.
- main.cpp - запуск Experiment::runAll().

 5. Выводы
1. Все методы имеют кубическую сложность, время работы сопоставимо. Выбор ведущего элемента даёт небольшие накладные расходы.
2. При решении многих систем с одной матрицей LU-разложение даёт многократный выигрыш (до 34 раз при k = 100) за счёт однократного разложения.
3. На плохо обусловленной матрице Гильберта методы без выбора ведущего элемента теряют устойчивость (при n ≥ 10). Частичный выбор сохраняет работоспособность, но точность падает с ростом n.
